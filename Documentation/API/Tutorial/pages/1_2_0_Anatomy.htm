RowManager Anatomy

Now lets take a look at how you build a RowManager.

Since a RowManager's job is to manage a database table, we will create an example table to manage. Let's suppose we want to create an application to track our favorite TV episodes. For the sake of simplicity, we only want to track a list of Episode names. The table would be something like this:

tvaddicts_episodes
episode_id episode_title

Note:

Some comments about naming conventions in how we design our tables:

Table Names:

Since all the tables in our site are stored in the same sitewide database, it helps to have meaningful names to our tables. So we generally follow this convention for naming tables: [applicationName]_[tableName]

  • [applicationName] = the name of the application/module you are creating.
  • [tableName] = the name of the table.

In our above example: [applicationName] = tvaddicts and [tableName]=episode

 

Column Names:

The columns in a table are named as follows: [tableName]_[columnValue]

The primary key is always: [tableName]_id

When a foreign key is included in a table, it keeps it's original name from the table where it is a primary key. For example:

tvaddicts_episodes
episode_id season_id episode_title
