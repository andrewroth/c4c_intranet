Modules

Now that you have a full fledged application that tracks your TV Addiction, we will see how to make your application plug into the supplied framework as a "Module".

When we are done, you will no longer need the controller scripts to drive your application, the framework will do that for you.

In order for us to understand how modules work, we need to first understand how our PageObject operates.

PageObject:

In our previous discussion of AppControllers, we talked about the differences between PageDisplay Objects and the AppController Object:

A PageDisplay object is primarily responsible for answering "What data needs to be displayed on this page?" However a PageDisplay object doesn't know about other PageDisplay objects, or when they should be displayed.

An AppController Object is primarily responsible for deciding "Which page in my application is to be displayed now?" It knows about all the different pages in an application and when each should be displayed. In this way, the AppController maintains the state of your application.

The Page Object is primarily responsible for deciding "Which Application is to be displayed now?" It knows about all the applications installed on the site, and is able to direct the viewer to the proper application. In addition, it also manages some of the security features of making sure a viewer is properly logged in before allowing them to view an application.

Now would be a good time to browse through the code of the site's index.php and Page.php:

index.php:

This script now simply loads the site objects, creates a new Page object, then runs the Page->start() method. All the rest of the processing is controlled by the start() method.

 

Page.php:

Most of the real work done here is done in the start() method. The ultimate goal of this method is to figure out which of the site's AppControllers to load, load it, drive it through the loadData(), processData() and prepareDisplayData() methods, then create the site template and generate the output to send back to the viewer's browser. It is much like the final controller script that you had in your previous examples.
