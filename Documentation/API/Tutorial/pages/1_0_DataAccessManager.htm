Data Access Managers

In many beginning php programming tutorials, they teach you to access data in database tables by writing code such as the following:

$sql = 'UPDATE table SET field1="New Info" WHERE index='.$rowID;
mysql_query( $sql );

However, in a large application, if you have hardcoded sql statements like this spread through out many files, it can become a nightmare to maintain if you need to make changes to the structure of your table.

In our framework, we want to seperate out the code for accessing and working with data in databases into seperate objects. We call these objects "Data Access Managers."

By containing all the code for interacting with your database tables in a single object, you minimize the amount of work needed to make changes to this table. If you do need to make changes to this db table, you just need to update this one object, and the rest of your code continues to function normally.

This part of the tutorial will introduce you to one of the most commonly used Data Access Managers: the RowManager.
