RowManager - Build Your Own

Now it is time to get your hands dirty and start writing your own RowManagers.

We will start by creating our own tvaddicts application. This application will manage 3 tables:

Series Table:

tvaddicts_series
series_id series_title
int (11) varchar(50)

To manage the different tv series we waste our lives away watching faithfully

 

Season Table:

tvaddicts_season
season_id series_id season_num
int(11) int(11) int(1)

To keep track of the different seasons of each series.

 

Episodes Table:

tvaddicts_episode
episode_id season_id episode_title episode_airDate episode_numberTimesWatched
int(11) int(11) varchar(50) date int(2)
(2 byte int ... we are addicts you know)

To track the specific episodes we watched, the first time they were aired, and the number of times we watched it.

 

What To Do

Use the attached template to create 3 new RowManagers to manage these tables. Save each file in your testing directory as SeriesManager, SeasonManager, and EpisodeManager. The template has several tags that should make it easy to tailor the data to your RowManager:

  • [TableName] : the name of your table with a Capital Letter (Episode)
  • [description] : a description of the purpose of this row manager
  • [author] : your name
  • [application] : the name of your application (tvaddicts)
  • [tableName] : the name of your table with lowercase first letter (episode)

Be sure to fill in the specific DB description constants.

Then add methods for getting and setting the individual fields.

Once you have then created, then copy the test_CreateTables.php file to your testing directory and run it in your browser: http://localhost/~accountName/siteDirectory/testDirectory/test_CreateTables.php

Once all the errors are debugged, and it runs successfully, then look in the DB to find your new tables.

Continue on to the next step.
