PageDisplay Objects - Page Flow

One of the biggest mental shifts about using our framework is getting used to grouping your code into 3 distinct phases: Loading Data, Processing Data, and Displaying HTML.

Loading Data:

In this phase, your job is to attempt to load into memory, create any necessary objects, pull from the Database, any data you might need to carry out the actions of this web page. It isn't that you can't initialize a variable in other phases, but you want to make sure you get the big items here. If you want to initialize a piece of data, and are asking yourself, "where do I put this?" then put it in this phase.

Processing Data:

This is the phase where you are actually doing the work of your page: updating db tables, crunching numbers, sending emails, etc...

Displaying HTML:

This is the final phase where you generate the HTML output that displays the result of your page.

In a previous example, you began to see the organization of the sample scripts into these phases. Now we are going to take these sample scripts, and formally put these phases into specific functions that drive these phases.
